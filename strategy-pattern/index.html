Strategy Pattern

Avinta Putri Agistasari

Hello, this is the first pattern that I've learn.
Definition :
"A family of algorithms, encapsulates each one, and makes them interchangeable. Strategy lets the algorithm vary independently from client that use it."

In other word, you can add behaviours dynamically.

This is an example of strategy pattern implementation.
Firstly, make a interface for the behaviour.
RunBehaviour.java 

public interface RunBehavior {  
    public void run();
}

ShootBehaviour.java 

public interface ShootBehavior {  
    public void shoot();
}

Then make concrete classes that implement RunBehaviour and ShootBehaviour interface. The concrete classes it depend on the implementation of behaviour that you want to add. For example, I want to make the A troop can run with foots or use skateboard but the B troop just run with foots.
This is the example of some concrete classes.
RunWithFoot.java 

public class RunWithFoot implements RunBehavior {

    @Override
    public void run() {
        System.out.println("Run with foot");
    }

}

RunWithSkateBoard.java 

public class RunWithSkateboard implements RunBehavior{

    @Override
    public void run() {
        System.out.println("Run with skateboard");
    }

}

ShootWithHandgun.java 

public class ShootWithHandgun implements ShootBehavior{

    @Override
    public void shoot() {
        System.out.println("Shoot with handgun");
    }

}

ShootWithSniper.java 

public class ShootWithSniper implements ShootBehavior{

    @Override
    public void shoot() {
        System.out.println("Shoot with sniper");
    }

}

And now, I make the Troops abstract class that will delegate its run and shoot behaviour.
Troops.java 

public abstract class Troops {

    RunBehavior runBehavior;
    ShootBehavior shootBehavior;

    public Troops(){}

    public abstract void attack();

    public void performShoot(){
        shootBehavior.shoot();
    }

    public void performRun(){
        runBehavior.run();
    }

    public void setRunBehavior(RunBehavior runBehavior) {
        this.runBehavior = runBehavior;
    }

    public void setShootBehavior(ShootBehavior shootBehavior) {
        this.shootBehavior = shootBehavior;
    }
}

Then create the concrete class of Troops that will implement the run and shoot behaviour depend on their abilities.
NewbieTroops.java 

public class NewbieTroops extends Troops{

    public NewbieTroops() {
        shootBehavior = new ShootWithHandgun();
        runBehavior = new RunWithFoot();
    }

    @Override
    public void attack() {
        System.out.println("Newbie troops start attacking");
    }

}

AdvancedTroops.java 

public class AdvancedTroops extends Troops{

    public AdvancedTroops() {
        shootBehavior = new ShootWithSniper();
        runBehavior = new RunWithSkateboard();
    }

    @Override
    public void attack() {
        System.out.println("Advanced troops start attacking");
    }

}

And lastly, lets run it!
TroopsSimulator.java 

public class TroopsSimulator {  
    public static void main (String [] args){
        Troops newbie = new NewbieTroops();
        newbie.attack();
        newbie.performRun();
        newbie.performShoot();

        Troops advanced = new AdvancedTroops();
        advanced.attack();
        advanced.performRun();
        advanced.performShoot();
        advanced.setShootBehavior(new ShootWithHandgun());
        advanced.performShoot();
    }

}

As you can see in the simulator class, you can add the behaviour of the advance troop dynamically and in runtime!
